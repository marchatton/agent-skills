# Changelog Draft

## Inputs

- Release scope and key changes.
- User-facing impact.
- Known limitations.

## Outputs

- `docs/release/<slug>/changelog.md`.

## Steps

1. Summarize user-visible changes.
2. Note fixes, improvements, and risks.
3. Include any migration or rollout notes.

## Verification

- Changelog covers impact, changes, and required notes.
