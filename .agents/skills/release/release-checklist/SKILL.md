# Release Checklist

## Inputs

- Release scope and risks.
- Rollout and rollback constraints.
- Monitoring requirements.

## Outputs

- `docs/release/<slug>/release.md`.

## Steps

1. Document rollout steps and rollback plan.
2. List monitors, alerts, and owners.
3. Capture preconditions and comms notes.

## Verification

- Checklist includes rollout, rollback, monitors, and post-release checks.
