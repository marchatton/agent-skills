# Create PRD

## Inputs

- Problem statement and goals.
- Target users.
- Constraints and risks.

## Outputs

- `docs/05-prds/<slug>.md`.

## Steps

1. Fill PRD template with scope, goals, and non-goals.
2. Define acceptance criteria and verification plan.
3. Capture risks and open questions.

## Verification

- Acceptance criteria are testable and mapped to verification steps.
