# Breadboard

## Inputs

- Feature goal.
- Actors and systems.
- Key constraints.

## Outputs

- `docs/05-prds/<slug>_breadboard.md`.

## Steps

1. List actors and systems.
2. Map the happy-path flow.
3. Note edge cases and risks.

## Verification

- Flow covers primary actors, systems, and edge cases.
