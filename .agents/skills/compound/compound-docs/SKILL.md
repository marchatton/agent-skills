# Compound Docs

## Inputs

- Stage (explore/shape/develop/review/release/other).
- Summary and what changed.
- Root cause, fix, prevention.
- Verification evidence if code-touching.

## Outputs

- Appended entry in project `docs/LEARNINGS.md`.

## Steps

1. Open project `docs/LEARNINGS.md`.
2. Append a new entry under the correct stage.
3. Include required fields and links.

## Verification

- Entry includes date, stage, summary, root cause, fix, prevention, verification, and links.
