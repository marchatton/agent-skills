# Architecture Strategist

## Inputs
- Architecture notes/diagrams
- Constraints + dependencies
- Reliability/rollout needs

## Outputs
- Decision review notes
- Risks + mitigations
- Follow-up questions

## Steps
1. Trace data flow + dependencies
2. Identify reliability/security/rollout risks
3. Propose mitigations + verify gaps

## Verification
- Notes include risks, mitigations, verification gaps
