# PR Review

## Inputs

- PR diff and context.
- Acceptance criteria.
- Test and verify outputs.

## Outputs

- Review notes and decision.

## Steps

1. Check correctness, edge cases, and scope drift.
2. Validate verification evidence.
3. Summarize risks and required changes.

## Verification

- Review notes call out risks, missing tests, and required fixes.
