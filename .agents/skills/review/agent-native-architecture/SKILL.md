# Agent-Native Architecture Review

## Inputs

- Architecture notes or diagrams.
- System constraints and dependencies.
- Reliability and rollout requirements.

## Outputs

- Architecture review notes.

## Steps

1. Review data flow and dependencies.
2. Identify reliability, security, and rollout risks.
3. Note missing observability or verification steps.

## Verification

- Review notes include risks, mitigations, and verification gaps.
