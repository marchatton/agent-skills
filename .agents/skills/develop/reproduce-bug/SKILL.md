# Reproduce Bug

## Inputs

- Bug report or signal.
- Environment details.
- Expected vs actual behavior.

## Outputs

- Repro steps and evidence.
- Minimal reproduction notes.

## Steps

1. Define expected vs actual behavior.
2. Capture deterministic repro steps.
3. Collect logs, screenshots, or recordings.

## Verification

- Repro steps produce the bug reliably.
